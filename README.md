1) O commit permite registrar ações e alterações, permitindo um controle para informar o que e quando alguma coisa foi alterada.
2) É interessante ter histórico, pois ajuda no monitoramento e gestão do projeto. O histórico revela informações importantes que foram registradas.
3) O branch, que é conhecido como "ramificação" ajuda o desenvolvimento paralelo ao permitir que vários desenvolvedores de um grupo tenham uma cópia própria. Isso auxilia no ato de reduzir o risco de alterações indesejadas, se caso forem revladas pelo commit.

1) Os repositórios remotos, fornecem agilidade e ferramentas importantes para o desenvolvimento em projetos colaborativos. Ou seja, é muito mais ágil e adequado para um desenvolvimento e execução com eficácia.
2) O git pull ajuda a sincronizar seu trabalho local com os de outros membros da equipe instantaneamente em repositórios remotos.
3) Porque o push irá salvar a versão atual do código. Se acontecer alguma falha de hardware ou qualquer outra eventualidade, o código estará salvo e seguro sem a perca permanente.
